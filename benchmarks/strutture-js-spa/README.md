# strutture-js SPA benchmark fixtures

This folder preserves real SCA SPA benchmark fixtures generated from the
ABP Ingegneria website UI and keeps a local runner for future `strutture-js`
performance checks.

## Contents

- `cases.json`: original SPA/UI project state plus three real cases.
- `models.generated.json`: expanded dump generated by the SPA adapter runner.
  This file is intentionally versioned here because it lets this repository run
  repeatable checks without depending on the website source tree.
- `adapter-entry.js`: original minimal SPA adapter export used by the website
  benchmark.
- `run.mjs`: original SPA-side runner. It requires the website repo layout
  because it imports `src/components/app/sca/...` adapter modules.
- `run-public-models.mjs`: local runner for this repository. It revives public
  or dump-derived RC section models and measures the core workflows available in
  `strutture-js`.

## Local use in this repository

```bash
npm run benchmark:spa-real-cases -- --list
npm run benchmark:spa-real-cases -- --iterations 10 --warmup 3
npm run benchmark:spa-real-cases -- --case sca-tbeam-biaxial-domain-public --json
```

The local runner currently measures:

- `sca-tbeam-biaxial-domain-public`, using the public workflow model generated
  by the SPA adapter;
- `sca-rect-service-stress-crack-width`, reconstructed from the dumped section,
  materials and SPA analysis state, then run through the public service-stress
  workflow;
- `sca-rect-service-deflection`, reconstructed through the SCA deflection
  adapter, which generates the synthetic service-moment profile required when
  the SPA dump has no FEM `analysisResult`.

## SPA-side runner

The original website-side runner is kept as `run.mjs`. Run it from
the website repository, or copy/symlink this folder back under that repo's
`benchmarks/` directory:

```bash
node benchmarks/strutture-js-spa/run.mjs --list
node benchmarks/strutture-js-spa/run.mjs --iterations 50 --warmup 10
node benchmarks/strutture-js-spa/run.mjs --dump-models benchmarks/strutture-js-spa/models.generated.json
```

That runner resolves `strutture-js` from the website repo's `node_modules`, so
it remains useful for before/after comparisons when the website is linked
against a local build of this package.
